# wc2026.cool 用户增长与传播优化建议

> 巡检时间：2026-06-13（Asia/Shanghai）
> 范围：阅读 `docs/HANDOFF.md`，查看生产站 `https://www.wc2026.cool/` 的首页、比赛详情、概率页、第三名计算器、串关、排行榜、观赛指南与 OG 图。本文只给优化建议，未改动现有网站代码。

## 一句话判断

当前站点已经有差异化资产，但增长入口没有把资产放到用户最想转发的瞬间。

最大机会不是做“又一个世界杯赛程站”，而是把用户此刻的焦虑和炫耀欲做成可参与、可截图、可转发的闭环：

1. “我的队还有戏吗？”第三名出线计算器。
2. “我比朋友更懂球”免注册预测挑战。
3. “这场爆冷后谁最受影响？”赛后 15 分钟内的摆动图卡。

## 当前现状

### 已经做对的地方

- 匿名即玩、免注册，是非常强的低摩擦优势。
- 第三名出线计算器抓住了 2026 新赛制下最强的解释型需求。
- 概率页有完整模型、夺冠概率、晋级概率、第三名形势，信任基础不错。
- 比赛详情页有 AI 前瞻和冷热门看点，比纯表格更有球迷语境。
- 合规边界已经有意识地处理：虚拟积分、不可兑换、免责声明、敏感词控制。
- 中英双语与海外华人定位明确，和 AdSense 变现路径匹配（注：Fubo 联盟 6/13 被拒且禁止再申请，已从变现路径剔除）。

### 影响关注量的主要问题

- 首页首屏更像“比赛列表”，不是“事件入口”。用户第一眼不知道为什么现在要停留、为什么现在要玩。
- 最强差异化工具“第三名出线计算器”藏在页脚和概率页 CTA，未进入底部主导航，也不是首页第一行动。
- 计算器可互动，但缺少结果闭环：没有“选择我的队/我的组”、没有“生成结论”、没有“一键复制/分享图卡”。
- 预测玩法有低门槛，但没有新用户 10 秒内的引导：初始积分、匿名开始、预测后会得到什么，首屏都不够明确。
- 串关页加载后可用，但生产巡检时所有选项显示 3.00，用户会感觉像占位数据，刺激感和可信度都弱。
- OG 图是品牌卡，不是结果卡。用户更愿意转发“我算出了某队还有戏/某队被挤出线”这类个人化结果。
- 排行榜已有数据，但缺少“挑战朋友”“晒战绩”“今日榜首”等社交触发。
- 英文增长面仍不足：部分关键页面和游戏文案硬编码中文，英语用户能被 metadata 引进来，但留存理由不完整。
- 缺少增长数据埋点。现在难以判断用户卡在首页、计算器、预测提交还是分享前。

## 用户视角的理想路径

建议把产品体验压缩成三个时间尺度：

- 3 秒：用户知道这个站能解决什么问题。
- 10 秒：用户完成一次无需注册的互动。
- 30 秒：用户得到一个值得发给朋友的结果。

推荐主路径：

1. 用户从社媒进入首页或某队链接。
2. 首屏看到“我的队还有戏吗？30 秒算第三名出线”。
3. 选择球队或小组，改几场结果。
4. 页面立刻生成一句结论：“如果日本胜瑞典，日本暂进前 8，瑞典跌到第 9。”
5. 用户点击“复制战报”或“生成分享图”。
6. 分享内容带回流链接：`wc2026.cool/calculator?...`
7. 回流用户再进入预测/排行榜/观赛指南。

## P0：下一步最该做的优化

### 1. 重做首页首屏信息架构

目标：让用户一进来就知道“现在要点哪里”。

建议首页首屏从比赛列表改为三入口结构：

- 主入口：第三名出线计算器。
  - 文案方向：“我的队还有戏吗？30 秒算出线”
  - CTA：“选择球队开始计算”
- 次入口：今日预测挑战。
  - 文案方向：“不用注册，先预测一场”
  - CTA：“预测今日焦点战”
- 变现入口：观赛指南。
  - 文案方向：“今晚比赛在哪看”
  - CTA：“看直播渠道”

比赛列表保留，但下移为“今日赛程”。首页不是少放内容，而是先给用户一个行动理由。

### 2. 把第三名计算器升为核心导航

当前底部导航是“预测 / 串关 / 概率 / 排行 / 我的”。从增长优先级看，建议短期把“串关”让位给“出线”或“计算器”。

理由：

- 计算器是差异化和社交传播资产。
- 串关目前更像重度玩法，且 3.00 的初始展示削弱信任。
- 小组赛末轮前，用户最强需求不是多场连选，而是“我的队怎样才能晋级”。

可选方案：

- 小组赛期间：预测 / 出线 / 概率 / 排行 / 我的。
- 淘汰赛后：预测 / 概率 / 排行 / 我的 / 观赛。

### 3. 计算器改成“先选我的队”

当前计算器一次展开大量小组，信息量很完整，但用户需要自己找队。

建议新增一个顶层选择：

- 选择球队。
- 或选择小组。
- 或从热门队快捷进入：美国、加拿大、墨西哥、巴西、阿根廷、日本、韩国、英格兰、法国、西班牙、葡萄牙、德国。

选择后默认只展开相关小组，并在顶部固定展示一句结论：

- “当前：韩国暂列 A 组第 2，晋级概率 95%。”
- “如果捷克胜南非、墨西哥平韩国，A 组第三将进入最佳第三名第 8。”

完整 12 组表可以折叠在下面，给深度用户使用。

### 4. 做分享闭环，而不是只做页面

最优先补两个轻功能：

- 复制文本：
  - “我刚算了一版：如果 X 胜 Y，A 组第三会排到最佳第三名第 8。你也来改一版：链接”
- 生成分享图：
  - 标题：我的队还有戏吗？
  - 主体：球队、当前排名、关键条件、最佳第三名分界线。
  - 底部：wc2026.cool，免费，无需注册。

第一版不必做复杂图片编辑器。可以先做固定模板：

- 第三名形势图。
- 某队命运图。
- 赛后摆动图。
- 个人战绩图。

### 5. 调整敏感玩法表述

从用户增长和广告合规看，建议逐步弱化“实时倍率 / 串关”的第一印象，换成更娱乐化的积分语言：

- “实时倍率”改成“积分奖励”或“社区奖励”。
- “串关”在中文 UI 可保留但降低首屏权重，英文避免高风险词。
- 所有 3.00 初始状态要解释为“社区初始奖励”，否则像假数据。

比赛详情页可以把重点从数字切到人气：

- “大家怎么看：主胜 42% / 平 25% / 客胜 33%”
- “冷门挑战：选少数派，命中得更多积分”
- “还没人预测这场，来当第一个”这个文案很好，可以继续强化。

### 6. 补增长埋点

没有埋点时，增长优化容易靠感觉。建议至少记录这些事件：

- `home_primary_cta_click`
- `calculator_team_selected`
- `calculator_pick_changed`
- `calculator_share_copy`
- `calculator_share_image`
- `match_prediction_selected`
- `prediction_submitted`
- `combo_leg_selected`
- `leaderboard_viewed`
- `watch_link_clicked`

核心漏斗：

- 访问首页 -> 点击计算器 -> 改结果 -> 复制/生成分享 -> 回流访问。
- 访问比赛页 -> 选择结果 -> 提交预测 -> 查看我的战绩 -> 次日回访。

### 7. 先修技术体验里的增长阻塞

这些不是纯工程洁癖，而是直接影响用户信任和传播：

- 按 `docs/HANDOFF.md` 的 P0，尽快补生产 `THE_ODDS_API_KEY`，避免概率页长期降级。
- 做 uptime 监控，至少监控 `/`、`/forecast`、`/calculator`、`/leaderboard`。
- 给 `/forecast` 预热缓存，避免首次 8 秒影响分享回流。
- 如果串关继续保留在主导航，必须让奖励数字看起来真实，或明确这是社区初始值。

## P1：6/22-6/27 小组赛末轮增长打法

官方赛程显示本届赛事从 2026-06-11 开始，决赛在 2026-07-19，小组赛末段会集中爆发“谁还能出线”的解释需求。站点应围绕这几天做内容生产。

### 场景型页面

为每组/每队生成可分享入口：

- `/calculator?group=A`
- `/calculator?team=kor`
- `/forecast/group/A`
- `/match/[id]?from=scenario`

页面标题按用户问题写：

- “韩国怎样才能出线？”
- “美国小组第三还有戏吗？”
- “最佳第三名第 8 名分界线现在是谁？”

### 赛后 15 分钟图卡

每场结束后快速生成：

- 本场结果。
- 小组排名变化。
- 最佳第三名分界线变化。
- 受影响最大的 3 支球队。
- “自己改下一场结果”的链接。

这类内容适合 X、Reddit、微信群、Telegram、WhatsApp。Reddit 只发 OC 数据图和计算器，不提竞猜玩法。

### 每日固定栏目

建议每天固定三张可分发素材：

- 今日焦点战：最影响出线的一场。
- 最危险第三名：第 8 与第 9 的分界。
- 爆冷摆动榜：哪场冷门会改变最多球队命运。

这些内容比泛泛“今日赛程”更有转发点。

## P2：增强留存和社交关系

等 P0/P1 跑通后，再做这些：

- 好友私有联赛/邀请房间。
- Wordle 式战绩分享：最近 5 场预测用色块展示。
- 成就分享卡：冷门猎手、神准、连续签到。
- Brier 准确率公开页，建立模型信任。
- 英文路由和 hreflang，补齐英语用户体验。
- 观赛指南和比赛详情打通：临近开赛的比赛页提示“在哪看”。

## 建议暂缓

- 暂缓大规模 SEO 长文。赛事窗口短，SEO 在 7/19 前很难成为主增长引擎。
- 暂缓复杂私有联赛。没有分享回流前，私域功能容易变成空房间。
- 暂缓强化串关为主入口。它不是当前最强差异化，且表述和合规成本更高。
- 暂缓面向大陆流量的大量投放。交接文档已说明大陆流量对当前变现路径贡献有限。

## 推荐落地顺序

1. 首页首屏改为“出线计算器 + 今日预测 + 观赛指南”三入口。
2. 底部导航把“出线/计算器”提到主导航。
3. 计算器增加“先选球队/小组”和顶部结论条。
4. 计算器增加复制分享文案。
5. 增加第一版结果图卡。
6. 修正或解释 3.00 初始奖励显示。
7. 补埋点和 uptime 监控。
8. 6/22 前准备小组赛末轮分发图卡模板。
