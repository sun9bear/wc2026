## 1. 项目概述与目标

做一个**纯娱乐**的足球预测社区网站。用户用每天免费领取、**无任何现实价值**的虚拟积分，对 2026 年世界足球大赛的比赛进行预测下注（胜平负、猜比分等），赢取积分、冲排行榜、解锁段位与成就，并把战绩分享给朋友。

**四大体验目标**（贯穿所有功能设计）：
- **参与感** —— 用户的预测能实时影响"倍率"和排行榜，赛中有强反馈。
- **成就感** —— 段位、命中率、连胜、徽章、排名，可见的成长与炫耀资本。
- **趣味** —— AI 趣味解说、冷热门、打脸墙、装饰收集，轻松好玩。
- **愿意分享** —— 战报卡、赛季总结卡、好友联盟邀请，自然病毒传播。

**非目标（明确不做）**：不涉及任何真实货币、不接入任何真实博彩公司数据、不提供任何真钱投注建议、不销售或导流任何彩票。详见第 2 节。

## 2. 合规边界与红线（最高优先级）

本产品合不合规，只取决于一件事：**有没有真实金钱或"有价值的东西"进出**。以下七条红线焊死在产品与代码中，任何功能、文案、AI 输出都不得触碰。

| # | 红线 | 实现要求 |
|---|---|---|
| 1 | **没有任何充值入口** | 积分只能通过游戏行为获得（注册赠送、每日签到、看广告补充、赢预测）；**不存在任何用法定货币购买积分的路径**。 |
| 2 | **没有任何提现 / 兑换出口** | 积分**不可**兑换为现金、实物、礼品卡、加密货币、其他平台货币或任何有价值之物。 |
| 3 | **积分不可在用户间转移 / 赠送** | 杜绝形成场外交易（私下用钱买卖积分）。 |
| 4 | **平台不抽水** | 不从预测输赢中抽取任何分成或佣金。 |
| 5 | **积分可免费无限补充 / 输光可重置** | 积分**无稀缺性**——这是削弱"虚拟筹码=有价值之物"法律认定（尤其美国 social-casino 诉讼）的关键设计。 |
| 6 | **措辞中性（去博彩化）** | 全站使用第 3 节术语表；绝不出现"充值/提现/兑奖/赌/庄家"等词；所有预测页带"仅供娱乐 · 积分无现实价值 · 不可兑换"声明（措辞见第 3 节强化规则）。 |
| 7 | **远离赛事方商标** | 不使用 "世界杯 / FIFA" 字样、会徽、奖杯、吉祥物；不使用 "中国体育彩票 / 竞彩" 官方名称与 logo。用中性表述（如"2026 年世界足球大赛""胜平负玩法"）。 |

**底层逻辑（务必理解）**：换中性词是"给已经干净的产品做品牌定位"，**不是**用来掩盖违法底层。只有在红线 1–5 守住（无钱无价值）的前提下，中性措辞才有意义；一旦引入金钱/价值，任何改名都无法免责，反而可能成为"主观规避"的证据。

**本项目明确规避的高危红线**（来自前期合规调研）：
- 跨境赌博 / 为赌博引流（→ 不接真实博彩赔率、不导流）。
- 非法售彩 / 彩票预测荐号（→ 不碰真钱彩票、不给真钱投注建议）。
- 国内备案/版号/微信审核（→ 受众与部署均在境外，见第 4 节）。

> 📌 **免责声明**：本节为基于公开信息的一般性合规设计，非法律意见。公开上线前应就目标市场（尤其美国/加拿大等海外华人聚集地）咨询当地执业律师。

## 3. 术语表（去博彩化措辞）

全站 UI、用户协议、营销文案、按钮文字、**以及 AI 生成内容**统一使用右列措辞。

| 禁用（博彩味） | 采用（趣味预测味） |
|---|---|
| 赔率 | **倍率**（亦可：趣味倍数 / 人气指数） |
| 下注 / 投注 | **预测** / 选边 |
| 赌注 / 注 / 投注金额 | **投入积分** |
| 投注单 | **我的预测 / 预测单** |
| 派彩 / 彩金 / 奖金 | **积分奖励 / 派分** |
| 中奖 | **命中 / 预测成功** |
| 庄家 | （池式无庄家，不使用） |
| 盘口 | **玩法 / 预测项** |
| 串关 / 过关 | **连串预测 / 连关挑战** |
| 让球 / 大小球 | **让分预测 / 进球数预测** |

整体词汇向 **预测 · 挑战 · 命中 · 积分 · 倍率 · 段位 · 战绩** 靠拢，彻底避开 **博彩 · 赌 · 彩票 · 竞彩 · 体彩 · 庄 · 盘**。

**⚠️ 强化规则（针对面向用户 / 审核器的公开文案）**
- **加否定词也不行**：公开文案（UI、商店描述、营销、广告承载页、AI 输出）中**绝不出现** `投注 / 下注 / 赌 / 博彩 / 彩票 / 竞彩 / 体彩 / 赔率 / 庄家 / 盘口 / 押注` 等词——**哪怕"非投注""不是赌博"这种否定写法也禁止**。关键词扫描器只匹配词、不看上下文，否定词救不了你，反而把雷词主动送进扫描结果。
- **免责声明改中性正向表述**：用「**仅供娱乐 · 积分无现实价值 · 不可兑换现金或实物**」来传达"非真钱"，**不要**写"非投注建议 / 不是赌博"。
- **唯一例外（私密 / 法律场景可明示）**：**苹果审核备注**与**用户协议 / 法律免责**中，可明确写"本产品不提供真钱博彩，积分无货币价值、不可购买或兑现"——这些不经广告关键词扫描，对人类审核者 / 律师把话说清反而有利。
- **范围说明**：本约束仅针对**公开文案**；本设计文档及代码注释属内部工程材料，为表述清晰可照常使用"下注 / 结算"等词。
- **多语言**：去博彩化术语表**每种语言各维护一份**——关键词扫描器按当地语言扫描（英文避 `bet/odds/wager/bookmaker`，用 `predict/multiplier/points/pick`；西语避 `apuesta/cuota`）。**新增任何语言前，必须先过一遍该语言的雷词表。**

## 4. 目标用户、平台与变现

- **目标用户**：海外华人足球爱好者（中文界面）。
- **平台形态**：响应式 Web 应用，**手机端与桌面端自适应**（移动优先）。
- **部署地**：境外（如 Vercel + Supabase 海外区）。因此**免**国内 ICP 备案、小游戏备案、版号、微信审核。

### 4.1 决策记录：受众 = 海外，国内市场不在范围内
经评估明确**只做海外华人市场**，**不**面向中国境内用户。原因：
- 国内化需 企业主体 + ICP 备案 +（可能）版号 + 内容审核；
- "竞猜/下注/倍率"形态即便无金钱，也过不了国内广告联盟（腾讯优量汇、穿山甲、百度联盟等）与平台的内容审核（赌博/竞猜类属明令禁止），需阉割成纯资讯/答题才行——核心玩法即失；
- 境外站面向国内用户还有被 GFW 屏蔽的风险。

> 「境外·免监管」与「在国内变现」互斥，只能择一；本项目选前者。Google 在墙内失效、国内广告联盟又需备案+企业+过审，故**国内广告投放不适用本产品**。若未来要做国内，属另起炉灶的重大变更。

### 4.2 变现方式（均为海外，且不触碰积分/输赢经济）
**核心原则**：可以赚钱，但只赚"与博彩机制无关"的钱——广告、去广告订阅、装饰、周边、打赏、非博彩联盟；**绝不**靠"卖积分 / 卖赢面 / 放价值出去"赚钱。

- **A. 广告（启动主力）**
  - 多广告网络分散：Google AdSense/AdMob + 备选 Media.net、Pangle 等，避免吊死一家（Google 可能因竞猜形态限流/拒号）。
  - 横幅 / 插屏。
  - **看广告补充积分**（激励式）：用户看广告免费领积分——无金钱流出用户、产生广告收入。
  - 直客品牌赞助 / 冠名（面向海外华人的品牌，CPM 更高）。
- **B. 非博彩电商 / 联盟**：球衣、观赛装备、EA FC、流媒体等联盟佣金；按需印刷周边。❌ 绝不接博彩公司联盟 / 导流。
- **C. 增值服务（⚠️ 只卖"不影响输赢"的东西）**：去广告会员；纯装饰 IAP（头像框/主题/皮肤）；联盟高级功能。⚠️ 绝不卖：积分、加注上限、续命/回血、专属高倍玩法（= 用钱买赢面，踩 social-casino 雷）。
- **D. 社区 / 自愿**：打赏/赞助（Ko-fi / 爱发电 / Patreon）；内容 / newsletter → 更多广告位。

**推荐节奏**：启动 = 广告（AdSense + 激励）+ 打赏 → 起量后 = 直客赞助 + 非博彩联盟 → 更晚（可选）= 去广告会员 + 装饰 IAP（支付与积分经济须**硬隔离**）。

**⛔ 变现红线（绝不做）**：博彩联盟/导流；用钱买积分或买赢面；给预测排行榜前几名发**真实奖品/奖金**（会触发抽奖/彩票/赌博认定）——奖励一律保持虚拟。

- **年龄**：建议设 **18+ 自我声明 + 年龄提示**（玩法形似竞猜，从严更稳妥）。
- ⚠️ 广告合规：Google 对博彩类内容有限制，故第 3 节中性措辞 + 免责声明同时服务于"过广告审核"。

### 4.3 多端策略与上架（Web 优先 → 安卓 → iOS）

**总原则**：Web 先行验证，App 分阶段跟进；统一用 **Capacitor / React Native 复用 Web 代码**做原生外壳，**严禁纯 WebView 套壳**（会被苹果 4.2「最小功能」驳回）。

| 阶段 | 平台 | 定位与要点 |
|---|---|---|
| 一 | **Web（响应式）** | 启动主战场：无审核门槛、最快、完全可控。优先把世界杯窗口吃下来。 |
| 二 | **安卓 App（Google Play）** | 政策比苹果宽松、相对好过；Capacitor/RN 复用 Web 代码 + 原生能力。注意 Google Play 在中国大陆不可用（对海外华人 OK），仍须遵守其模拟博彩政策与年龄分级。 |
| 三 | **iOS App（App Store）** | 风险最高、审核最严；Web/安卓验证后再做。务必做成真正原生体验，并满足下方苹果清单。 |

> App 外壳要加**推送 / 离线 / 原生 Tab 导航 / Widget** 等"网页给不了"的原生能力，才能过苹果 4.2，并提升留存。

**苹果 App Store 审核注意事项（清单）**
- ✅ **全程无真钱**（红线 1/2 天然契合）：不充值买积分、不提现兑奖。iOS 上任何数字商品（去广告会员、装饰）**必须走苹果 IAP**（抽成 30%/15%），广告变现不受影响。
- ⚠️ **死规则——玩法与奖励概念分离**：截图、描述、文案中**绝不能出现"玩游戏 → 赢奖品"的关联**，二者必须分开表述。
- ⚠️ **17+ 年龄分级**（频繁/强烈模拟博彩）。
- ⚠️ **元数据中性**：App 名/描述/截图不得暗示真实博彩或"赢钱"，第 3 节中性措辞延伸到商店页。
- ⚠️ **不接真实博彩赔率**（与第 2 节一致）。
- ⚠️ **上非中国区 App Store**：中国区游戏需版号、博彩形态基本禁；面向海外受众上美区等非中国区。
- 📝 **备好审核说明**：明确声明"无真钱、积分无现实价值、不可兑现/购买"；预期苹果审核主观、可能来回沟通。
- ⚠️ **过审 ≠ 零法律风险**：美国 social-casino 类诉讼风险（见第 11 节）独立存在，与商店审核无关。

> 安卓同理：Google Play 也有模拟博彩政策与年龄分级，但通常比苹果宽松；"无真钱"声明与中性元数据同样适用。

### 4.4 收款与税务（中国大陆个人发布者）
> 适用前提：运营者为中国大陆个人、产品部署境外、受众在海外。

- **可用个人账户**：Google AdSense 支持中国个人发布者注册，**无需公司**（区别于国内广告联盟需企业 + 备案）。
- **收款方式（2026-06 现状）**：
  - **电汇（EFT / Wire）**：美元到中国银行卡，$100 起付；招商银行外汇业务较顺、常可直接入卡（部分银行要求 Google 提供合同/证明）。
  - **PayPal Hyperwallet**：2025-10-30 起对中国发布者开放，每月 21–26 号到账；需**新建专用 Hyperwallet 账户**（不能复用现有 PayPal）。对个人比电汇更省心，推荐。
- **外汇**：个人年度结汇便利化额度等值 **5 万美元**。
- **GFW**：注册 / 管理 AdSense 需科学上网（Google 控制台墙内打不开）。
- **税务**：境外所得，**依法申报缴纳个人所得税**。
- **关键澄清**：发布者人在国内**不影响**赚海外用户广告费——广告对海外受众正常展示计费，钱通过电汇 / Hyperwallet 收回国内；"Google 广告墙内失效"只影响给国内用户展示，与发布者收款无关。
- **账号安全**：竞猜形态可能触发 AdSense 博彩内容审查，配合 §3 中性措辞 + 多广告网络分散（见 §4.2）降低封号风险。
- AdMob（未来做 App）同理支持中国发布者收款。

## 5. 产品功能

### 5.1 核心增长飞轮
```
预测下注 →【参与】倍率因你而动、排行榜实时跳
   → 结算见分晓【趣味/刺激】
   → 战绩↑ 段位↑ 徽章解锁【成就】
   → 一键生成炫耀战报卡【分享】
   → 朋友点进来、拉进好友联盟【参与+留存】
   → 回到预测下注 … 🔁
```

### 5.2 玩法菜单（按实现成本分层）

**Tier 1 · 单场玩法**（共用通用引擎，新增=填配置+一段结算规则，成本低）
- 胜平负（1X2）
- 让分预测（让球）
- 进球数预测（大小 / 区间）
- 猜比分（精确比分）
- 半全场
- 双方是否进球

**Tier 2 · 连串预测（串关）**：选多场、全部命中才赢、倍率连乘。需多腿下注与多腿结算逻辑（中等成本）。`bets` 表从一开始即设计为**可多腿**以原生支持。

**Tier 3 · 赛事级长线预测**（贯穿整届，结算节奏不同）
- 夺冠预测、小组出线、金靴射手
- **晋级树 Bracket**：填淘汰赛对阵表，逐轮兑现得分。

**Tier 4 · 社交 / 成长系统**（独立模块）
- **好友私人联盟**（建房间与好友单独比拼）
- 每日签到 / 连胜奖励、成就徽章、装饰兑换（用积分兑换头像框/称号，纯装饰、无价值，属健康积分回收）
- 分层排行榜（总榜 / 每轮 / 好友榜 / 各玩法榜）、个人战绩档案

### 5.3 体验目标 → 抓手

| 目标 | 抓手 |
|---|---|
| 参与感 | 倍率因你的预测实时浮动；赛中排行榜实时跳动、结算瞬间动画；"本场 68% 押 A，你是少数派"；每日签到/连胜；Web 推送提醒（开赛/结算）。 |
| 成就感 | 段位系统（按命中率/积分晋级）；战绩档案（命中率、最高连胜、历史最高排名）；徽章（冷门猎手、比分大师、五连胜…）；分层排行榜让更多人能登顶某榜。 |
| 趣味 | AI 拟人趣味前瞻+赛后吐槽；冷热门；预测理由；打脸墙/名人堂；积分兑换装饰。 |
| 愿意分享 | 可炫耀**战报卡**（押中绝杀自动生成漂亮图片）；赛季总结卡；好友联盟邀请奖励（双方各得积分，无金钱）。 |

> 💡 **分享设计要点**：海外华人重度使用微信，而**微信会拦截外部链接**。因此分享主打**生成图片卡**（图片不被拦截），靠朋友圈/小红书自然扩散；好友邀请用**邀请码 + 二维码**而非纯链接。

| 层 | 选型 | 说明 |
|---|---|---|

**备选并否决**：自建 Node/Express + 自管 Postgres —— 单人 + 紧迫工期下基建成本过高，否决。

### 6.1 国际化（i18n）与多语言
- **从第一天 i18n-ready**：用 next-intl（或 next-i18next），按 locale 路由（`/zh` `/en` `/es`…），所有文案抽到语言文件，**禁止硬编码字符串**。
- **语言节奏**：首发 **中文 + 英文**；快速跟进 **西班牙语 + 葡萄牙语**（拉美/巴西/西班牙足球票仓）；再视情况加 阿拉伯语（注意 RTL 布局）、法、日、韩。
- **本地化**：日期/数字/时区按 locale；AI 内容按语言分别生成。
- ⚠️ **合规延伸**：去博彩化术语表**每语言一份**（见 §3）；多语言把受众从"海外华人"扩展到**全球足球用户**，但合规姿态不变（海外、无金钱、各语言中性措辞、18+、非中国区）。

### 6.2 账号策略与反滥用
**原则：对"人"宽松，对"机器 / 小号"有轻防护。** 无金钱 → 无需 KYC / 实名 / 身份证 / 支付信息。

- **宽松前门**：游客即玩（匿名，数据存本地）→ 一键升级为账号（社交 OAuth / 邮箱）以留存战绩、上榜。Supabase Auth 原生支持 匿名 + 社交 + 邮箱。
- **轻量反滥用后端**（保护变现与公平，不加身份负担）：
  - 设备级限流 + 人机验证（reCAPTCHA / Cloudflare Turnstile）+ "一设备一号"启发式；
  - 邮箱 / 手机验证**仅在关键环节**触发（登全球榜、领激励奖励、开"看广告补积分"）；
  - **广告无效流量防护**——假号刷激励广告 = invalid traffic，可致 **AdSense 封号**，是头号风险，必须防。
- **18+ 年龄确认**：出生日期 / 勾选，轻门槛但必须有（竞猜形态 + 苹果 17+ 分级）。

## 8. 核心机制

### 8.1 倍率算法（池式众包）
倍率反映"群众怎么押"，营造动态盘感，但**不构成需要守恒的真实资金池**——因为积分无价值且无限。

- 某选项 s 的倍率：
  `multiplier_s = clamp( (P + k·N) / (S_s + k), 下限=1.10, 上限=可配置(如 50) )`
  其中 `P` = 该玩法总投入积分，`S_s` = 选项 s 累计投入，`N` = 选项数，`k` = 平滑系数（给每个选项一个虚拟底注，避免除零与早期极端值）。
- **下注时锁定倍率**：用户预测瞬间记录 `multiplier_at_bet`；倍率随后续投入实时浮动（参与感）。
- **结算**：命中者获得 `投入积分 × 锁定倍率`，未命中者失去投入积分。**无需保证池子守恒**（积分无价值、可无限增发），故不会出现"派分超过池子"的真实风险。
- 可选：叠加一条**中性统计胜率**参考线（用公开排名/Elo 自算），标注"赛事分析参考"，纯编辑内容。

### 8.2 结果与结算流
```
赛程：静态预置（赛程提前公布、固定）
结果：定时任务拉免费足球数据 API → 与第二来源交叉校验
        ├─ 一致 → 自动结算该场所有玩法（按锁定倍率派分、写流水、刷新段位/排行榜/徽章）
        └─ 冲突 / 缺数据 / 改期弃赛 → 写 result_audit、推送提醒管理员 → 人工确认后结算
```
- AI Agent / 爬虫**仅用于"第二来源交叉校验"与内容生成**，不作为唯一结算依据（避免幻觉/解析错误污染结算）。
- **改期 / 弃赛 / 腰斩比赛**：对应玩法作废，已投入积分**全额退回**（`points_ledger` 记 `reason=refund`）；若发生在**串关**中，该腿倍率按 **1.0** 计、不影响其余腿（其余腿正常结算）。
- 因积分无价值，偶发结算错误为"体验问题"而非"金钱损失"，但仍以准确性优先（保留存）。

### 8.3 积分经济
- **获取**：注册赠送（如 1000）、每日签到、连胜奖励、看广告补充、命中派分。
- **消耗**：投入预测、兑换装饰（头像框/称号，纯外观）。
- **兜底**：余额过低可免费领取/重置（红线 5）。
- **审计**：每一笔变动入 `points_ledger`，服务端唯一可信源。

## 9. AI 内容层
- **用途**：赛前趣味前瞻、数据解读、冷热门提示、赛后小结、拟人解说。
- **定位**：看球助兴 / 游戏内趣味参考，**严禁**输出"投注建议 / 推荐下注 / 必赢"等。
- **约束**：系统提示词中注入第 3 节术语表 + 强制结尾"仅供娱乐 · 积分无现实价值 · 不可兑换"（不得出现"投注"等词，即使否定形式）；输出入库缓存，定时生成，降低成本与延迟。

### 9.1 运营与内容自动化（AI Agent 协同）
单人运营 + 多语言 + 赛期高频更新 → 必须 AI 深度参与。模型：**AI 干量、人管异常与风险决策，外加一个统一审核台**——自动化处理重复劳动，人只碰需判断的部分。

**日常分工**

| 工作 | AI / 自动化负责 | 人（运营者）只管 |
|---|---|---|
| 比赛数据 & 结算 | API + 第二源交叉校验 + 自动派分 | 处理冲突 / 改期异常 |
| 内容（前瞻 / 赛后 / 冷热门，全语言） | LLM 定时批量生成、入库缓存 | 抽查 + 审异常稿 |
| 盘口 / 玩法创建 | 从赛程按模板自动生成 | — |
| UGC 审核（昵称 / 理由 / 联盟名） | AI 分类器自动隐藏 / 标记 | 审边界 case |
| 社交 / 推送（战报卡、赛事提醒） | 自动生成图片卡 + 排期推送 | 定策略、把口径 |
| 运营监控（结算异常 / 作弊 / 报错 / 广告掉量） | 看门狗 Agent 汇总「今日待办」 | 看 digest 决策 |
| 客服 | FAQ + AI 客服 | 接罕见 / 复杂问题 |

**三条不可破的护栏**
1. **AI 不做结算事实源**：只起草内容 + 当看门狗；权威结算永远是 API + 人工兜底（防幻觉污染结算，与 §8.2 一致）。
2. **AI 公开输出先过"雷词 lint"再发布**：生成 → 每语言关键词过滤（+ 必要时二次 AI 校）→ 才发布；把 §3 文案红线**自动化**。
3. **不过度自动化脆弱 / 有 ToS 风险环节**：自动发帖可能违反社媒 ToS、自动审核会误判 → 保留人工抽查。

**成本策略**：激进缓存复用（生成一次多处用）；routine 内容用便宜模型、高曝光内容用好模型。

**关键基建**：一个轻量 **admin 控制台**，把"AI 草稿 / 异常 / 待审"全排到一处，运营者一键 通过 / 改 / 拒。目标日常运维 ≈ 每天约 30 分钟看 digest + 审标记项。运营者定位 = **主编 + 风控**，而非内容工 / 录入员。

> ⚠️ 这套管线 + 护栏 + 控制台本身是**前期开发量**，需在分波计划排期：建议 Wave 1 先上"内容生成 + 雷词 lint + 结算看门狗 + 最小 admin"，社交 / 客服自动化放后续波次。

## 10. 分波上线计划（对齐赛程）

> 工期现实：单人在开幕前完整上线全部功能不现实；赛事约 6 周，**分波铺开、一个不砍**。部分玩法本就该晚上（如晋级树需淘汰赛对阵明朗）。若 Wave 1 来不及，开赛后数天再上也无妨（绝大多数比赛在后面）。**宁可晚几天，也不上一个会算错结算的版本。**

| 波次 | 时间窗 | 内容 |
|---|---|---|
| **Wave 1 · 启动** | 小组赛开打前后 | 通用玩法引擎 + 登录/积分 + 单场 4 种（胜平负·让分·进球数大小·猜比分）+ 池式倍率 + 预测单 + 自动结算 + 夺冠预测 + 排行榜（总/每轮）+ 每日签到/连胜 + 段位 + 个人战绩 + **战报卡** + AI 前瞻 + 广告位 |
| **Wave 2 · 小组赛期** | 6 月中下旬 | 半全场 + 双方进球 + **连串预测** + **好友私人联盟** + 徽章 + 装饰兑换 + 邀请奖励 + 更多 AI 内容 |
| **Wave 3 · 淘汰赛前** | 6 月底 | **晋级树 Bracket**（赶上淘汰赛）+ 金靴等长线盘 + 赛季总结卡 + 打脸墙 + 打磨 |
