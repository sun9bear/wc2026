## 1. 设计原则

1. **暗色优先、硬朗有劲**：深色底 + 高对比 + 粗体大数字，适合男性体育/电竞审美，夜间观赛护眼。
2. **数据即英雄**：倍率、比分、命中率、段位等数字是视觉主角，用粗体大字号呈现。
3. **像游戏，不像赌场**：用游戏化的段位、勋章、XP 表达"竞技成就"，**禁用**筹码、扑克、轮盘、钞票等博彩意象。
4. **移动优先**：先设计 375–414px，再向上自适应到桌面（多列）。
5. **克制的霓虹**：强调色点到为止（电光绿为主），避免满屏彩虹显得廉价/俗艳。
6. **中年也读得清**：正文 ≥14px、对比达标、信息别过密（见 §8）。

## 2. 色彩

### 2.1 基础色板（Design Tokens）

| Token | HEX | 角色 |
|---|---|---|
| `--bg` | `#0A0E13` | 页面背景（最底层） |
| `--surface` | `#141A22` | 卡片 / 容器表面 |
| `--surface-2` | `#1B232D` | 次级表面（输入、选项、行） |
| `--border` | `#283340` | 描边 / 分隔线 |
| `--text` | `#E8EDF2` | 主文字 |
| `--muted` | `#8A97A6` | 次要文字 / 占位 |
| `--green` | `#1BE27F` | **主色**：主操作 / 命中 / 选中 / 正向 |
| `--green-dim` | `rgba(27,226,127,.14)` | 选中态底色 / 绿色微光 |
| `--blue` | `#2E9BFF` | 次强调：信息 / 链接 / 铂金段位 |
| `--amber` | `#FFB02E` | 平局 / 连胜 / 提醒 |
| `--red` | `#FF5436` | 客胜 / 热门 / 失利 / 危险 |
| `--gold` | `#F5C451` | 段位荣誉 / 领奖台 / 奖励高光 |

### 2.2 语义角色（用 token 表达意义，别直接堆颜色）
- **主操作 / 提交 / 成功 / 命中** → `green`
- **信息 / 次要操作 / 铂金** → `blue`
- **平局 / 连胜 / 黄金段位 / 待办提醒** → `amber`
- **失利 / 热门 / 倍率下跌 / 错误** → `red`
- **荣誉 / 排名 Top / 奖励** → `gold`
- **胜平负三选项**：选中态统一用 `green` 高亮（不靠红绿区分胜负，避免色盲歧义，靠"选中"状态 + 文案区分）。

### 2.3 合规约束（颜色与意象）
- ❌ 禁用：筹码红/赌场绿毡、扑克、轮盘、金币雨、钞票、骰子等**博彩视觉符号**。
- ✅ 采用：球场、球队色、电竞 HUD、段位勋章、数据面板等**体育/竞技符号**。

## 3. 字体排印

### 3.1 字族
| 用途 | 字体 | 说明 |
|---|---|---|
| 标题 / 数字 | **Oswald**（备选 Space Grotesk） | 粗体、紧凑、运动感；用于倍率、比分、积分、段位等大数字 |
| 正文 / UI | **Inter** | 清晰易读 |
| 中文 | **Noto Sans SC** | 与 Inter 搭配的 CJK 回退 |

> 多语言：拉丁字母走 Oswald/Inter；CJK 走 Noto Sans SC；阿拉伯语另配 RTL 字体（后续）。

### 3.2 字阶（移动端基准）
| 级别 | 字号 / 行高 | 字重 | 典型用途 |
|---|---|---|---|
| Display | 26–40 / 1.1 | 700 | 倍率、比分、关键大数字 |
| Title | 22–30 / 1.2 | 700 | 页面/品牌标题 |
| Heading | 15–18 / 1.3 | 600 | 区块小标题 |
| Body | 14–15 / 1.5 | 400–500 | 正文 |
| Caption | 11–12 / 1.4 | 500 | 次要说明、标签、免责声明 |

**规则**：所有"数字主角"（倍率/比分/积分/命中率/段位分）一律用标题字体 + 粗体，制造数据张力。

## 4. 间距 · 圆角 · 阴影

- **间距比例（4px 基准）**：`4 · 8 · 12 · 16 · 20 · 24 · 32`。卡片内边距 14–16；区块间距 16。
- **圆角**：`sm 8`（输入/选项）· `md 14`（按钮/小卡）· `lg 18`（比赛卡/大卡）· `pill 30`（标签/段位条）· `full`（头像/圆点）。
- **阴影与发光**：
  - 卡片：`0 30px 60px rgba(0,0,0,.55)`（仅大容器/手机框，列表卡可更轻）。
  - **绿色微光**（选中/高亮/CTA）：`0 0 18px rgba(27,226,127,.15)`，点到为止。
- **手机框参照**：宽 ~384px、圆角 34（仅样图展示用，真实页面用响应式容器）。

## 5. 组件规范

> 每个组件标注状态。所有交互元素 hover/active/focus/disabled 必须有可见反馈。

### 5.1 按钮 Button
- **Primary（主操作，如"提交预测"）**：`green` 实底、深色字 `#06231a`、`md` 圆角、绿色投影；按下轻微下沉。
- **Secondary**：透明底 + `border`，文字 `text`。
- **Ghost / 文字按钮**：仅文字 + 颜色态。
- 禁用：降透明度、去阴影、禁指针。

### 5.2 预测选项 Pick（胜平负等）
- 三列等宽卡片：上方小标签（主胜/平局/客胜）+ 下方**大倍率数字**。
- **默认**：`surface-2` 底 + `border`。
- **选中**：`green` 描边 + `green-dim` 底 + 绿色微光 + 倍率变绿。
- **趋势角标**：右上 ▲（`green`，倍率上升）/ ▼（`red`，下降）。
- 禁用（已封盘）：灰化 + 禁点击。

### 5.3 比赛卡 Match Card
- 顶部标签行：阶段/组别 + `LIVE 倍率`（绿点脉冲）。
- 队伍区：双队徽 + 队名 + 中间开赛时间/倒计时。
- 含人气条（§5.6）与预测选项（§5.2）。
- 圆角 `lg`、表面渐变（`#18222c → #131a22`）。

### 5.4 投入积分 Stake
- 数字（标题字体）+ 滑杆（绿色填充 + 白色拨钮 + 绿色光环）。
- "命中可派分 +X"：`green` 高亮数字。

### 5.5 段位 / 徽章 Badge & Rank
- **段位条 chip**：pill 圆角，按层级配色——王者/钻石 `blue`、黄金 `gold`、白银 银灰、铂金 `blue`。图标用盾牌 🛡。
- **成就徽章**：方卡 + emoji/图标 + 名称；**未解锁**：灰度 + 降透明度。
- **领奖台**：Top1 金环 + 皇冠 + 放大；Top2/3 银/铜环。

### 5.6 数据可视 Bars
- **人气条**：分段条（绿/琥珀/红）表示三选项占比 + 文案百分比。
- **XP 进度条**：绿→蓝渐变填充 + "x / y XP"。

### 5.7 排行榜行 Leaderboard Row
- 名次（标题字体）+ 头像 + 昵称 + 段位 chip + 积分 + 涨跌 ▲▼。
- **当前用户行**：`green` 描边 + `green-dim` 底高亮。

### 5.8 底部标签栏 Tab Bar
- 4 项：预测 / 排行 / 战绩 / 我的；图标 + 文字；选中态 `green`。
- 顶部 `border` 分隔，背景比页面略深。

### 5.9 战报卡 Share Card
- 绿色渐变边框卡，含战绩文案 + "生成"按钮（`green` pill）。
- 生成的是**图片**（绕开微信链接拦截，见方案 §5.3）。

### 5.10 免责声明 Disclaimer（合规组件）
- 固定文案：**「仅供娱乐 · 积分无现实价值 · 不可兑换」**（Caption 级、`muted` 色）。
- 出现在每个预测页底部；**不得**包含"投注/赌/博彩"等词（含否定形式，见方案 §3）。

## 6. 动效 Motion

| 场景 | 效果 | 时长/缓动 |
|---|---|---|
| LIVE 圆点 | 脉冲呼吸 | 1.6s 循环 |
| 倍率变动 | 数字闪一下 + ▲▼ 染色 | 200ms ease-out |
| 数字入场 | 计数滚动（count-up） | 400–600ms |
| 选中选项 | 描边 + 微光渐显 | 150ms |
| 结算命中 | 高光/庆祝（克制，不刺眼） | 300–500ms |
| 排行榜变动 | 行重排平滑过渡 | 250ms |

- 统一缓动 `ease-out`；交互反馈 ≤200ms。
- **必须**遵守 `prefers-reduced-motion`：用户开启则关闭非必要动画。

## 7. 图标与图像
- 图标：线性、统一笔重；底栏可用填充态表达选中。
- 队徽：MVP 用 emoji 国旗/占位圆形，后续替换为**中性来源**的真实队徽（注意版权，见方案 §11）。
- 不使用任何赛事方（世界杯/FIFA）官方标识（方案红线 7）。

---

## 8. 无障碍 Accessibility
- **对比度**：正文/重要文字对背景达 WCAG AA；`muted` 仅用于次要信息。
- **点击目标** ≥ 44×44px（移动端）。
- **焦点可见**：键盘焦点用 `green` 焦点环。
- **字号**：正文 ≥14px（兼顾中年用户）。
- **动效**：尊重 `prefers-reduced-motion`。
- **色彩不单独承载信息**：胜负/状态除颜色外配文案或图标（色盲友好）。

---

## 9. 文案与措辞（与合规挂钩）
- 全部 UI 文案遵循方案 **§3 术语表 + 强化规则**：用 倍率/预测/投入积分/派分/命中/段位/战绩；**禁用** 投注/下注/赌/博彩/赔率/庄家/盘口（**含否定形式**）。
- 免责声明用 §5.10 的中性正向文案。
- **多语言**：每种语言维护一份去博彩化术语表；新增语言先过雷词表（方案 §6.1）。
- **年龄**：18+ 提示（方案 §6.2）。

## 10. 实现：Tailwind 映射

在 `tailwind.config` 中以 CSS 变量接入 token，便于将来扩展浅色模式：

```js
// tailwind.config.js (摘要)
theme: {
  extend: {
    colors: {
      bg:'var(--bg)', surface:'var(--surface)', 'surface-2':'var(--surface-2)',
      border:'var(--border)', text:'var(--text)', muted:'var(--muted)',
      green:'#1BE27F', blue:'#2E9BFF', amber:'#FFB02E', red:'#FF5436', gold:'#F5C451',
    },
    fontFamily: {
      head:['Oswald','Noto Sans SC','sans-serif'],
      body:['Inter','Noto Sans SC','sans-serif'],
    },
    borderRadius:{ sm:'8px', md:'14px', lg:'18px', pill:'30px' },
    boxShadow:{
      card:'0 30px 60px rgba(0,0,0,.55)',
      glow:'0 0 18px rgba(27,226,127,.15)',
    },
  },
}
```
```css
/* globals.css — 暗色为默认主题 */
:root{
  --bg:#0A0E13; --surface:#141A22; --surface-2:#1B232D; --border:#283340;
  --text:#E8EDF2; --muted:#8A97A6;
}
```
